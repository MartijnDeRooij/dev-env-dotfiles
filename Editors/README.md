# Editors

This document covers the editor ecosystem used throughout this repository.

The goal is to maintain a keyboard-driven workflow across Linux and Windows + WSL2.

---

## ℹ️ Editor Overview

| Category | Tool |
|-----------|------|
| Multiplexer | [.tmux.conf](terminal/.tmux.conf) |
| Primary Editor | [Neovim](terminal/README.md) |
| IDE | [VSCode](terminal/README.md) |
| Alternative Editor | [Emacs](terminals/README.md) |

---

## 📝 Neovim

### Why Neovim?

- Keyboard-driven workflow
- Fast startup
- Language Server Protocol support
- Treesitter syntax highlighting
- Portable configuration

---

## 💻 VSCode

### Recommended Usage

VSCode is primarily used when:

- Deep debugging is required
- Remote development is needed
- Working with teams standardized on VSCode

---

## 🧠 Emacs

### Why Emacs?

Emacs is more than an editor and can function as a complete computing environment.

Supported setups:

- Vanilla Emacs
- Doom Emacs
- Spacemacs
- Emacs From Scratch
